# Security Policy

## Supported Versions

Currently, as the project is in active pre-v1.0 development, only the latest
release is actively supported with security updates.

| Version  | Supported          |
| -------- | ------------------ |
| Latest   | :white_check_mark: |
| < Latest | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub
issues.**

If you discover a security vulnerability within the Markdown BDD Transpiler,
please send an email directly to the maintainer at:
**binkley@alumni.rice.edu**

### What to include in your report

- A description of the vulnerability and its potential impact.
- Steps to reproduce the vulnerability.
- Any proposed solutions or mitigations, if you have them.

### What to expect

- We will acknowledge receipt of your vulnerability report as soon as
  possible.
- We will investigate the issue and determine if a patch is required.
- We will keep you informed of our progress towards a fix.
- Once the issue is resolved, we will publish a security advisory and credit
  you for the discovery (unless you prefer to remain anonymous).
